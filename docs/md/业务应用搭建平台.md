# 2. 产品背景

## 2.1 背景说明

企业在日常经营中会产生大量内部系统建设需求，例如：

* 客户管理
* 项目管理
* 合同管理
* 采购审批
* 请假报销
* 工单流转
* 设备巡检
* 库存管理
* 数据统计
* 报表分析

传统开发模式通常需要经过需求沟通、原型设计、数据库建表、接口开发、前端开发、测试、部署等流程，开发周期长，修改成本高。

业务应用搭建平台通过可视化配置方式，让业务人员、实施人员或开发人员快速搭建业务应用，降低重复开发成本，提高企业内部系统建设效率。

# 5. 使用场景

## 5.1 客户管理 CRM

销售人员维护客户信息、跟进记录、商机阶段、合同金额。
管理者可以查看销售漏斗、客户转化率、成交金额等指标。

## 5.2 项目管理

项目经理创建项目，分配任务，跟踪进度，记录问题和风险。
团队成员可以提交任务进展，负责人查看项目统计报表。

## 5.3 审批管理

企业内部的请假、报销、采购、合同、付款等流程都可以通过流程设计器配置。

## 5.4 工单管理

适用于 IT 运维、售后服务、设备报修、客户问题处理等场景。
工单可按类型、优先级、负责人自动分配和流转。

## 5.5 设备巡检

企业可以配置巡检表单、巡检计划、异常上报、处理流程和统计报表。

## 5.6 库存 / 进销存管理

支持商品档案、入库、出库、库存盘点、库存预警、供应商管理等基础能力。

## 5.7 数据统计与经营看板

企业管理者可以通过报表仪表盘查看销售、项目、工单、库存、审批等业务指标。

# 6. 产品目标

## 6.1 MVP 目标

第一阶段完成业务应用搭建平台的核心闭环：

1. 可以创建应用
2. 可以创建数据模型
3. 可以配置表单字段
4. 可以生成列表页、详情页、编辑页
5. 可以配置基础审批流程
6. 可以进行数据新增、修改、删除、查询
7. 可以配置角色、菜单、按钮和数据权限
8. 可以导入导出 Excel
9. 可以发布应用并给业务用户使用

## 6.2 中长期目标

1. 支持报表仪表盘
2. 支持 API 集成
3. 支持自动化工作流
4. 支持应用模板
5. 支持 AI 生成表单、流程、SQL
6. 支持移动端 H5
7. 支持私有化部署和 SaaS 化部署

# 8. 核心功能需求

## 8.1 应用管理

### 8.1.1 功能说明

应用是业务应用搭建平台中的业务系统单位。一个应用可以包含多个数据模型、表单页面、流程、菜单和权限配置。

### 8.1.2 功能列表

| 功能   | 说明                  |
| ---- | ------------------- |
| 创建应用 | 设置应用名称、图标、描述、分类     |
| 编辑应用 | 修改应用基础信息            |
| 删除应用 | 删除未发布或无效应用          |
| 复制应用 | 复制已有应用作为新应用         |
| 发布应用 | 将应用发布给用户使用          |
| 停用应用 | 暂停应用访问              |
| 应用模板 | 从模板创建应用             |
| 应用分类 | CRM、OA、工单、项目、库存、巡检等 |

---

## 8.2 数据建模

### 8.2.1 功能说明

数据建模用于定义业务对象和字段，相当于平台的动态建表能力。

例如：

* 客户表
* 合同表
* 项目表
* 任务表
* 工单表
* 商品表
* 库存表
* 巡检记录表

### 8.2.2 支持字段类型

| 字段类型 | 说明               |
| ---- | ---------------- |
| 单行文本 | 姓名、标题、编号         |
| 多行文本 | 备注、描述、说明         |
| 数字   | 金额、数量、分数         |
| 日期   | 年月日              |
| 日期时间 | 精确到时分秒           |
| 单选   | 多个选项选一个          |
| 多选   | 多个选项选多个          |
| 下拉选择 | 下拉框选择            |
| 人员选择 | 选择系统用户           |
| 部门选择 | 选择组织部门           |
| 附件   | 上传文件             |
| 图片   | 上传图片             |
| 子表   | 一条主数据下包含多条明细     |
| 关联表  | 关联其他业务表          |
| 公式字段 | 根据其他字段自动计算       |
| 系统字段 | 创建人、创建时间、更新时间、状态 |

### 8.2.3 字段配置能力

| 配置项    | 说明             |
| ------ | -------------- |
| 字段名称   | 业务展示名称         |
| 字段编码   | 系统内部字段编码       |
| 字段类型   | 文本、数字、日期、人员等   |
| 是否必填   | 控制提交时是否必填      |
| 默认值    | 固定值、当前用户、当前时间等 |
| 校验规则   | 长度、范围、格式、正则    |
| 是否唯一   | 例如客户编号、合同编号    |
| 是否可查询  | 是否作为搜索条件       |
| 是否可排序  | 是否支持排序         |
| 是否列表展示 | 是否在列表页展示       |

---

## 8.3 表单设计器

### 8.3.1 功能说明

表单设计器用于通过拖拽方式生成业务录入页面。

### 8.3.2 页面结构

| 区域    | 说明              |
| ----- | --------------- |
| 左侧组件区 | 展示可拖拽字段和布局组件    |
| 中间画布区 | 表单设计区域          |
| 右侧属性区 | 配置字段属性、样式、校验、事件 |
| 顶部操作区 | 保存、预览、发布、撤销、重做  |

### 8.3.3 核心能力

| 功能   | 说明               |
| ---- | ---------------- |
| 拖拽组件 | 通过拖拽快速设计表单       |
| 字段绑定 | 表单组件绑定数据模型字段     |
| 分组布局 | 支持分组、卡片、Tab、栅格   |
| 字段校验 | 必填、格式、长度、范围校验    |
| 默认值  | 支持当前用户、当前部门、当前时间 |
| 显隐规则 | 根据字段值动态显示或隐藏     |
| 只读规则 | 根据状态、角色控制是否可编辑   |
| 子表单  | 支持明细数据录入         |
| 表单预览 | 保存前预览真实效果        |
| 表单发布 | 发布后供业务用户使用       |

---

| 报表页  | 图表和统计分析    |

### 8.4.3 列表页配置

| 功能   | 说明             |

## 8.5 流程设计器

### 8.5.1 功能说明

流程设计器用于配置企业内部审批和业务流转。

### 8.5.2 流程节点类型

| 节点类型 | 说明              |
| ---- | --------------- |
| 开始节点 | 流程发起            |
| 审批节点 | 指定人员、角色或部门负责人审批 |
| 抄送节点 | 通知相关人员          |
| 条件分支 | 根据字段值进入不同流程路径   |
| 会签节点 | 多人全部同意后通过       |
| 或签节点 | 任意一人同意后通过       |
| 服务节点 | 调用接口或执行系统动作     |
| 结束节点 | 流程结束            |

### 8.5.3 审批功能

| 功能   | 说明           |
| ---- | ------------ |
| 提交审批 | 用户提交表单后进入流程  |
| 同意   | 审批通过并流转到下一节点 |
| 驳回   | 退回发起人或上一节点   |
| 撤回   | 发起人撤回流程      |
| 转交   | 审批人转交给其他人    |
| 加签   | 临时增加审批人      |
| 抄送   | 通知相关人员       |
| 审批记录 | 查看完整流程历史     |
| 超时提醒 | 超过处理时间自动提醒   |

---

## 8.6 权限中心

### 8.6.1 功能说明

权限中心用于控制不同用户能访问哪些应用、页面、按钮和数据。

### 8.6.2 权限类型

| 权限类型 | 说明                  |
| ---- | ------------------- |
| 应用权限 | 控制用户是否能访问某个应用       |
| 菜单权限 | 控制用户是否能看到菜单         |
| 页面权限 | 控制用户是否能访问页面         |
| 按钮权限 | 控制新增、编辑、删除、导入、导出等按钮 |
| 字段权限 | 控制字段是否可见、是否可编辑      |
| 数据权限 | 控制用户可以查看哪些数据        |
| 流程权限 | 控制用户能否发起、审批、撤回流程    |

### 8.6.3 数据权限规则

| 规则       | 说明               |
| -------- | ---------------- |
| 全部数据     | 可查看所有数据          |
| 本部门数据    | 只能查看本部门数据        |
| 本部门及下级数据 | 查看本部门和下级部门数据     |
| 仅本人数据    | 只能查看自己创建的数据      |
| 自定义数据范围  | 按字段、角色、组织进行自定义过滤 |

## 8.9 API 集成中心

### 8.9.1 功能说明

API 集成中心用于对接企业已有系统，实现数据获取、数据推送和接口调用。

### 8.9.2 功能列表

| 功能      | 说明                      |
| ------- | ----------------------- |
| 接口管理    | 新增、编辑、删除接口              |
| 请求方式    | GET、POST、PUT、DELETE     |
| 鉴权配置    | Token、Header、Basic Auth |
| 参数配置    | Query、Path、Body、Header  |
| 返回值解析   | JSONPath 提取返回字段         |
| 参数映射    | 表单字段映射到接口参数             |
| 接口测试    | 在线测试接口是否可用              |
| 调用日志    | 记录请求参数、响应结果、耗时          |
| 失败重试    | 接口失败后自动重试               |
| Webhook | 支持外部系统推送数据              |

### 8.9.3 典型场景

| 场景          | 说明              |
| ----------- | --------------- |
| 表单提交后调用接口   | 例如提交订单后推送到 ERP  |
| 流程审批通过后调用接口 | 例如采购审批通过后生成采购单  |
| 定时同步数据      | 每天同步客户、订单、库存数据  |
| 报表绑定接口      | 图表直接使用外部 API 数据 |

## 8.10 自动化工作流

### 8.10.1 功能说明

自动化工作流用于实现“当某个事件发生时，自动执行某些动作”。

### 8.10.2 触发器

| 触发器   | 说明               |
| ----- | ---------------- |
| 数据新增时 | 某张表新增数据后触发       |

| 定时触发  | 每天、每周、每月定时执行     |
| 接口触发  | 外部 Webhook 调用后触发 |

### 8.10.3 动作

| 动作   | 说明             |
| ---- | -------------- |
| 修改数据 | 自动更新某张表数据      |
| 新增数据 | 自动创建业务记录       |
| 发送通知 | 站内信、短信、邮件、企业微信 |
| 调用接口 | 请求第三方 API      |
| 发起流程 | 自动发起审批流程       |
| 生成报表 | 自动生成统计结果       |

# 21. 简历项目包装建议

## 项目名称

**企业级业务应用搭建平台**

## 项目描述

该项目是一个面向企业内部业务场景的业务应用搭建平台，支持通过可视化方式完成数据建模、表单设计、页面配置、流程审批、权限管理、数据管理、报表分析和 API 集成。平台基于 Spring Boot、Vue3、Flowable、MyBatis-Plus、Redis 等技术实现，能够帮助企业快速搭建 CRM、OA 审批、工单管理、项目管理、设备巡检、库存管理等内部业务系统，减少重复 CRUD 开发，提高业务系统交付效率。

## 技术栈

Spring Boot、Spring Cloud Alibaba、MyBatis-Plus、Flowable、Redis、MySQL、PostgreSQL、Vue3、Element Plus、ECharts、BPMN.js。

## 个人职责

1. 负责业务应用搭建平台整体架构设计，包括应用管理、数据建模、表单配置、页面配置、流程配置和权限体系设计。
2. 设计并实现元数据驱动的数据模型能力，支持字段配置、动态表单渲染、列表查询和数据导入导出。
3. 集成 Flowable 流程引擎，实现流程部署、流程发起、待办处理、审批驳回、流程历史查询等功能。
4. 设计 RBAC 权限模型，支持菜单权限、按钮权限、字段权限和数据权限控制。
5. 设计 API 集成中心，支持外部接口配置、参数映射、接口调用和调用日志记录。
6. 参与报表仪表盘模块设计，支持业务数据统计、图表展示和经营看板。
7. 负责系统部署方案设计，支持企业本地化部署和 Docker 部署。

## 项目亮点

1. 基于元数据驱动实现动态表单和动态页面，减少大量重复 CRUD 开发。
2. 集成 Flowable 流程引擎，支持企业常见审批和业务流转场景。
3. 支持角色、部门、字段、按钮、数据范围多维度权限控制。
4. 支持 API 集成能力，可与 ERP、CRM、OA、财务系统等外部系统对接。
5. 支持报表和经营看板扩展，帮助企业快速完成业务数据统计分析。
6. 后续可扩展 AI 生成表单、AI 生成流程、AI 问数等智能化能力。
