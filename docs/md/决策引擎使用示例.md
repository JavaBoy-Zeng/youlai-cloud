# 决策引擎使用示例

本文说明当前决策引擎 MVP 如何使用，并给出一个完整的“交易风控”示例。

## 1. 当前能力边界

当前决策引擎已经支持一条完整主链路：

1. 在系统菜单中进入“决策引擎”。
2. 配置场景、变量、规则、规则集和决策流。
3. 发布资产。
4. 通过实时决策接口传入业务参数。
5. 引擎返回决策结果、风险等级、分数、命中规则、执行路径和 traceId。
6. 执行过程写入执行日志，资产操作写入审计日志。

当前可执行的核心资产是：

- 场景：`SCENE`
- 规则：`RULE`
- 规则集：`RULE_SET`
- 决策流：`FLOW`

以下资产当前主要作为配置资产管理和后续扩展预留：

- 变量：`VARIABLE`
- 数据源：`DATA_SOURCE`
- 模型：`MODEL`
- 评分卡：`SCORE_CARD`
- 决策表：`DECISION_TABLE`

其中，决策流节点支持 `DATA`、`MODEL`、`SCORE_CARD`、`DECISION_TABLE` 类型真实执行。`DATA` 和 `MODEL` 支持 HTTP/静态输出/请求字段映射、超时、降级输出和本地短缓存；`CALCULATE` 节点仍保留 `outputs` 或旧 `mockOutput` 的兼容写法。

## 2. 菜单入口

后台菜单结构如下：

```text
首页
决策引擎
  场景
  变量
  规则
  规则集
  决策流
  数据源
  模型
  评分卡
  决策表
```

这些菜单来自 `youlai_system.sys_menu`，不是前端静态路由自动生成的。若菜单调整后页面未刷新，需要清理 Redis 菜单缓存或重新登录。

```bash
redis-cli -a 'foobared' del 'menu:routes'
```

## 3. 启动服务

建表脚本：

```bash
docs/sql/decision_engine.sql
```

启动决策引擎服务：

```bash
mvn -pl decision-engine -am spring-boot:run
```

如果通过网关访问，接口前缀是：

```text
http://localhost:9999/decision-engine/api/v1/decision-engine
```

如果直接访问决策引擎服务，接口前缀是：

```text
http://localhost:18080/api/v1/decision-engine
```

本文下面的示例默认使用网关地址。

## 4. 交易风控示例

业务目标：

当用户发起交易时，根据订单金额和交易城市给出风控决策。

规则如下：

- 订单金额大于 `10000`，进入人工审核，风险等级为 `HIGH`，加 `82` 分。
- 交易城市属于 `重庆` 或 `深圳`，进入观察，风险等级为 `MEDIUM`，加 `20` 分。
- 如果两条规则同时命中，引擎会合并更高优先级的决策结果和风险等级，分数累加，上限为 `100`。

### 4.1 场景

菜单位置：`决策引擎 / 场景`

新增场景：

```json
{
  "code": "trade_risk",
  "name": "交易风控",
  "category": "风控",
  "status": "草稿",
  "tags": "实时,交易,风控",
  "owner": "risk-team",
  "remark": "交易风控实时决策场景",
  "content": {
    "inputs": [
      { "field": "orderAmount", "type": "NUMBER", "required": true },
      { "field": "ip", "type": "STRING", "required": false },
      { "field": "deviceId", "type": "STRING", "required": false },
      { "field": "city", "type": "STRING", "required": false }
    ],
    "outputs": ["decisionResult", "riskLevel", "score", "hitRules", "traceId"]
  }
}
```

保存后点击“发布”，状态会变为 `已发布`。实时决策只会执行 `已发布` 或 `已启用` 状态的资产。

### 4.2 变量

菜单位置：`决策引擎 / 变量`

变量用于描述输入字段，当前不参与强校验，主要方便管理和后续扩展。

订单金额变量：

```json
{
  "code": "orderAmount",
  "name": "订单金额",
  "category": "基础变量",
  "status": "草稿",
  "tags": "交易,金额",
  "owner": "risk-team",
  "remark": "请求入参变量",
  "content": {
    "type": "NUMBER",
    "source": "REQUEST",
    "defaultValue": 0,
    "description": "本次交易订单金额"
  }
}
```

交易城市变量：

```json
{
  "code": "city",
  "name": "交易城市",
  "category": "基础变量",
  "status": "草稿",
  "tags": "交易,地域",
  "owner": "risk-team",
  "remark": "请求入参变量",
  "content": {
    "type": "STRING",
    "source": "REQUEST",
    "description": "本次交易所在城市"
  }
}
```

### 4.3 规则

菜单位置：`决策引擎 / 规则`

规则由两部分组成：

- `conditions`：命中条件。
- `actions`：命中后对决策上下文的影响。

大额交易规则：

```json
{
  "code": "RISK_001",
  "name": "大额交易转人工",
  "category": "交易风控",
  "status": "草稿",
  "tags": "高风险,审批",
  "owner": "risk-team",
  "remark": "订单金额超过一万进入人工审核",
  "content": {
    "sceneCode": "trade_risk",
    "priority": 100,
    "conditions": {
      "logic": "AND",
      "items": [
        { "field": "orderAmount", "operator": ">", "value": 10000 }
      ]
    },
    "actions": {
      "decisionResult": "REVIEW",
      "riskLevel": "HIGH",
      "score": 82,
      "tags": ["大额交易", "人工审核"],
      "reason": "订单金额超过 10000",
      "outputs": {
        "reviewQueue": "trade-risk"
      }
    }
  }
}
```

敏感城市规则：

```json
{
  "code": "RISK_002",
  "name": "敏感城市观察",
  "category": "交易风控",
  "status": "草稿",
  "tags": "观察,地域",
  "owner": "risk-team",
  "remark": "敏感城市交易进入观察",
  "content": {
    "sceneCode": "trade_risk",
    "priority": 60,
    "conditions": {
      "logic": "AND",
      "items": [
        { "field": "city", "operator": "in", "value": ["重庆", "深圳"] }
      ]
    },
    "actions": {
      "decisionResult": "OBSERVE",
      "riskLevel": "MEDIUM",
      "score": 20,
      "tags": ["地域观察"],
      "reason": "交易城市命中观察名单"
    }
  }
}
```

保存后分别发布两条规则。

### 4.4 规则集

菜单位置：`决策引擎 / 规则集`

规则集用于把多条规则编排到一个场景下执行。

```json
{
  "code": "trade_risk_default_set",
  "name": "交易风控默认规则集",
  "category": "交易风控",
  "status": "草稿",
  "tags": "规则集,默认",
  "owner": "risk-team",
  "remark": "按优先级执行交易风控规则",
  "content": {
    "sceneCode": "trade_risk",
    "strategy": "ANY",
    "shortCircuit": false,
    "ruleCodes": ["RISK_001", "RISK_002"]
  }
}
```

字段说明：

- `sceneCode`：绑定场景编码。
- `strategy`：当前支持 `ANY` 和 `ALL` 语义标识；`ALL` 不满足时会在 `outputs.ruleSetStrategy` 写入 `ALL_NOT_SATISFIED`。
- `shortCircuit`：命中一条规则后是否短路停止。
- `ruleCodes`：规则执行顺序；未配置时按规则优先级降序执行。

保存后发布规则集。

### 4.5 决策流

菜单位置：`决策引擎 / 决策流`

如果某个场景存在已发布决策流，引擎会优先执行决策流；否则按场景执行规则集或规则。

```json
{
  "code": "trade_risk_flow",
  "name": "交易风控决策流",
  "category": "交易风控",
  "status": "草稿",
  "tags": "决策流,实时",
  "owner": "risk-team",
  "remark": "开始、规则集、结束的演示决策流",
  "content": {
    "sceneCode": "trade_risk",
    "nodes": [
      { "type": "START", "code": "start", "sort": 1, "enabled": true },
      { "type": "RULE_SET", "code": "trade_risk_default_set", "sort": 2, "enabled": true },
      { "type": "END", "code": "end", "sort": 3, "enabled": true }
    ]
  }
}
```

保存后发布决策流。

## 5. 接口调用示例

### 5.1 实时决策

```bash
curl -X POST 'http://localhost:9999/decision-engine/api/v1/decision-engine/decision/execute' \
  -H 'Content-Type: application/json' \
  -d '{
    "sceneCode": "trade_risk",
    "eventId": "EVT202606190001",
    "userId": "10001",
    "params": {
      "orderAmount": 12000,
      "ip": "127.0.0.1",
      "deviceId": "D001",
      "city": "重庆"
    }
  }'
```

预期返回结构：

```json
{
  "code": "00000",
  "data": {
    "traceId": "TRACE...",
    "eventId": "EVT202606190001",
    "sceneCode": "trade_risk",
    "decisionResult": "REVIEW",
    "riskLevel": "HIGH",
    "score": 100,
    "tags": ["大额交易", "人工审核", "地域观察"],
    "hitRules": [
      {
        "ruleCode": "RISK_001",
        "ruleName": "大额交易转人工",
        "reason": "订单金额超过 10000",
        "score": 82,
        "decisionResult": "REVIEW",
        "riskLevel": "HIGH",
        "tags": ["大额交易", "人工审核"]
      },
      {
        "ruleCode": "RISK_002",
        "ruleName": "敏感城市观察",
        "reason": "交易城市命中观察名单",
        "score": 20,
        "decisionResult": "OBSERVE",
        "riskLevel": "MEDIUM",
        "tags": ["地域观察"]
      }
    ],
    "path": [
      "SCENE:trade_risk",
      "FLOW:trade_risk_flow",
      "START:start",
      "RULE_SET:trade_risk_default_set",
      "RULE:RISK_001=true",
      "RULE:RISK_002=true",
      "END:end"
    ],
    "outputs": {
      "reviewQueue": "trade-risk"
    },
    "elapsedMs": 12
  },
  "msg": "一切ok"
}
```

说明：

- `decisionResult` 按优先级选择更高结果：`PASS < OBSERVE < REVIEW < REJECT`。
- `riskLevel` 按优先级选择更高等级：`LOW < MEDIUM < HIGH`。
- `score` 会累加命中规则分值，上限为 `100`。
- `path` 记录执行路径，方便排查策略是否按预期执行。

## 6. 条件配置说明

条件支持嵌套 `AND` / `OR`。

```json
{
  "logic": "OR",
  "items": [
    { "field": "orderAmount", "operator": ">", "value": 10000 },
    {
      "logic": "AND",
      "items": [
        { "field": "city", "operator": "in", "value": ["重庆", "深圳"] },
        { "field": "deviceId", "operator": "is not null" }
      ]
    }
  ]
}
```

支持的操作符：

| 操作符 | 说明 |
| --- | --- |
| `=` / `eq` | 等于 |
| `!=` / `ne` | 不等于 |
| `>` / `gt` | 大于 |
| `>=` / `gte` | 大于等于 |
| `<` / `lt` | 小于 |
| `<=` / `lte` | 小于等于 |
| `in` | 在集合内 |
| `not in` / `not_in` | 不在集合内 |
| `contains` | 字符串包含 |
| `between` | 闭区间 |
| `regex` | 正则匹配 |
| `is null` | 为空 |
| `is not null` | 非空 |

字段支持点路径取值，例如：

```json
{
  "field": "user.profile.age",
  "operator": ">=",
  "value": 18
}
```

如果入参是：

```json
{
  "user": {
    "profile": {
      "age": 20
    }
  }
}
```

则该条件会命中。

## 7. 前端操作建议流程

推荐按以下顺序配置：

1. 新增并发布场景。
2. 新增变量，用于描述入参字段。
3. 新增规则，并用“策略测试 / 规则测试”验证单条规则。
4. 新增规则集，把多条规则按顺序组织起来。
5. 新增决策流，把规则集挂到流程节点中。
6. 用“策略测试 / 实时决策”验证完整链路。
7. 查看“执行日志”和 `traceId`，确认执行路径和命中规则。

## 8. 常见问题

### 8.1 为什么实时决策提示资产未发布或未启用？

实时决策只会执行状态为 `已发布` 或 `已启用` 的资产。保存资产后需要点击“发布”。

### 8.2 为什么改了菜单但前端没有变化？

菜单来自后端 `sys_menu` 和 Redis 缓存。修改菜单后需要清理：

```bash
redis-cli -a 'foobared' del 'menu:routes'
```

然后刷新页面或重新登录。

### 8.3 为什么数据源、模型、评分卡、决策表配置后没有参与执行？

当前 MVP 中，这些资产主要用于统一管理和后续扩展。实际执行链路目前以规则、规则集、决策流为主。

### 8.4 空库启动后为什么已经有交易风控示例？

`DemoDataInitializer` 会在 `decision_artifact` 为空时初始化一套交易风控演示数据，并自动发布。

如果不需要演示数据，可以删除初始化数据或移除该初始化器。
